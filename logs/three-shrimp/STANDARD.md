## 二、任务分配标准流程

### 阶段1：需求接收（来福虾主导）
```
[Frank 提出需求]
    ↓
[来福虾] 需求澄清 → 明确目标、交付物、截止时间
```

### 阶段2：任务拆解与分配（来福虾执行）
来福虾必须在30秒内判断任务类型，并决定以下之一：

| 任务类型 | 分配方案 | 示例 |
|---------|---------|------|
| 单虾任务 | 直接指定一只虾执行 | "扣子虾，画一张海报" |
| 双虾协作 | 明确主负责+协作方 | "扣子虾画图，来福虾写文案" |
| 三虾并行 | 启动标准并行流程 | 见下方"三虾并行执行流程" |

### 阶段3：三虾并行执行流程（标准版）
```
[来福虾] 在群内 @所有相关虾，发布任务卡片
    ↓
┌─────────────────────────────────────────────────────┐
│  🚀 各虾并行执行（明确分工）：                        │
│  ① 来福虾 → 中文资料、框架搭建、进度监控、成果整合   │
│  ② Q福虾 → 海外数据、本地文件、Gemini查询            │
│  ③ 扣子虾 → 可视化、代码、配图、影片                 │
└─────────────────────────────────────────────────────┘
    ↓
[各虾] 完成后回复：【XX虾 完成】+ 成果摘要
    ↓
[来福虾] 汇总融合 → 统一输出 → 交付Frank
```

### 阶段4：成果交付（来福虾负责）
- 检查完整性（是否所有子任务完成）
- 统一格式（字体、排版、文件命名）
- 交付给Frank并确认验收

## 四、日志记录规范

### 📁 日志系统结构
```
logs/three-shrimp/
├── README.md              # 系统说明（静态）
├── STANDARD.md            # 本规范文档（静态）
├── YYYY-MM-DD.md          # 每日主日志（每日唯一）
└── archive/               # 归档目录（每月迁移）
    └── 2026-03/
```

### 🎯 主文档定义（重要！）
**每一天只有一个主文档，命名为：`YYYY-MM-DD.md`**

| 规则 | 说明 |
|------|------|
| **唯一性** | 一天只有一个主文档，所有内容都写进这个文件 |
| **命名格式** | `2026-03-24.md`（年-月-日.md） |
| **位置** | `logs/three-shrimp/` 根目录下 |
| **创建者** | 来福虾（当日首次协作时自动创建） |
| **维护者** | 来福虾（实时更新，当日结束确认完整性） |

### 📝 主文档内容结构
```markdown
# 三虾协作日志 - YYYY-MM-DD

## 📋 今日概览
（日期、天气、在线虾、任务统计）

## 🎯 任务清单
（所有任务条目，按时间顺序）

## 💬 关键对话
（重要决策、问题讨论）

## 📎 附件索引
（各虾产生的独立文件列表）

## 📝 备注与待办
（遗留问题、明日计划）
```

### 🔄 内容写入规则
1. **群内对话** → 来福虾整理后写入"关键对话"章节
2. **任务执行** → 各虾回复完成状态时，来福虾更新"任务清单"
3. **独立文件** → 各虾产生的文件（如图表、代码）保留在原位置，仅在日志中记录索引
4. **当日结束** → 来福虾检查主文档完整性，确认无遗漏

### ⏰ 记录时机
- **任务启动**: 来福虾立即在主文档创建任务条目
- **任务完成**: 各虾回复后，来福虾2分钟内更新状态
- **对话发生**: 重要对话结束后，来福虾10分钟内整理入日志
- **每日结束**: 来福虾整理当日主文档，确认完整性

### 🗄️ 归档规则
- **保留期限**: 当前月份 + 上一个月保留在主目录
- **归档触发**: 每月1日，来福虾将两个月前的日志移至 `archive/YYYY-MM/`  
- **示例**: 4月1日，将2月及之前的日志移至 `archive/2026-02/`

### 🔍 如何查看历史
```bash
# 查看今天
cat logs/three-shrimp/$(date +%Y-%m-%d).md

# 查看昨天
cat logs/three-shrimp/$(date -d "yesterday" +%Y-%m-%d).md

# 查看指定日期
cat logs/three-shrimp/2026-03-24.md

# 搜索关键词
grep -r "天气测试" logs/three-shrimp/*.md
```

## 五、输出格式标准

### 📄 Markdown文件规范
```markdown
【XX虾 输出】

## 标题

### 二级标题

- 关键结论用 **加粗** 标注
- 数据用表格呈现

| 列1 | 列2 |
|-----|-----|
| 数据 | 数据 |

1. 有序列表用于步骤
2. 清晰、简洁

---
状态：已完成 / 待完成 / 需接力
```

### 🎨 可视化输出（扣子虾）
- 图片：PNG/JPG，文件名包含日期和主题
- 图表：注明数据来源、单位、时间范围
- 配色：优先使用浪潮蓝/橙或简洁商务风格

### 📊 数据输出（Q福虾）
- 表格：CSV格式便于来福虾整合
- 数据：注明来源、采集时间、可靠性
- 代码：提供可复现的脚本

## 六、容错与降级机制

### 🚨 单虾离线处理

| 离线虾 | 降级方案 | 执行者 |
|-------|---------|--------|
| Q福虾 | 海外搜索由来福虾用kimi-search替代 | 来福虾 |
| 扣子虾 | 可视化改为文字描述/表格 | 来福虾 |
| 来福虾 | 扣子虾临时统筹，Q福虾协助 | 扣子虾 |

### ⚠️ 任务超时处理
1. **超时15分钟**: 来福虾在群内询问进展
2. **超时30分钟**: 来福虾评估是否需要调整分工
3. **超时60分钟**: 来福虾向Frank汇报阻塞原因，申请延期或调整需求
