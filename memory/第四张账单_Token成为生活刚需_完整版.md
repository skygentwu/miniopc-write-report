## 从三表到四表

过去，每个家庭的墙上都挂着三只表：水表、电表、煤气表。

它们记录着你最基本的生存消耗——喝的水、用的电、烧的气。按月结算，按量付费，这是工业时代留给我们的生活方式。

现在，**第四张表**正在悄然进场。

它不是挂在墙上的实体表，而是运行在云端的无形计量器。它不记录你消耗了多少物理资源，而是记录你**调用了多少次智能**。

**这就是Token表。**

OpenClaw用户的"凌晨欠费事件"，只是这场变革的预演。

## Token是什么？简单理解为"智能的单位"

过去你用搜索引擎，免费。现在你用AI助手，按Token收费。

一次对话、一篇文案、一段代码、批量处理一百份文档——背后都是大模型在消耗算力和电力进行推理。这些推理的计算量，被折算成Token，成为可定价、可交易、可计量的数字商品。

**Token的本质，是智能服务的原子化计量单位。**

就像：
- 1度电 = 1千瓦时能量
- 1立方米水 = 1吨水
- **1个Token ≈ 0.75个英文单词的语义处理量**

OpenClaw这类Agent工具之所以容易导致"Token爆炸"，是因为它会**自动调用多个工具链**：搜索→阅读→总结→写作→格式化，一个任务可能触发几十次模型调用，累积起来就是天文数字。

## Token计费模式的底层逻辑

为什么Token会复制水电煤的成功？

**第一，边际成本趋近于零，但总量巨大。**

生成一个Token的边际成本极低（几分钱的千分之一），但当一个人一天消耗几万个Token时，就形成了稳定、可预测的消费需求。

**第二，不可储存，必须实时生产。**

就像电不能大规模储存，Token也必须在你提问的那一瞬间实时生成。这意味着供给侧必须保持"随时准备为你推理"的状态——这正是基础设施的特征。

**第三，需求刚性且持续增长。**

人类对智能的渴望永无止境。从"偶尔用用"到"时刻依赖"，Token消耗只会越来越多。这是典型的"成瘾性刚需"。
