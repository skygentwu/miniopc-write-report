# 🦐 双虾交接协议 v2.0

> 建立时间：2026-03-22
> 版本：v2.0
> 目的：解决双虾无法自动连续对话的问题

---

## 一、核心原则

**主动轮询 + 明确交接**

- 发起方：主动发起对话，主动检查对方回复
- 接收方：回复后主动声明"已完成，等待接力"
- 禁止：被动等待，必须主动检查 GitHub 文件

---

## 二、交接格式

### 发起方格式
```markdown
【双虾交接】
from：云虾/Q福虾
to：Q福虾/云虾
task：xxx
时间戳：HH:MM
@对方 请在5分钟内回复并继续
```

### 接收方回复格式
```markdown
【双虾回复】
状态：完成/进行中
内容：xxx
@对方 已回复，请接力继续或检查
```

---

## 三、轮询机制

### 主动检查规则
1. 发起对话后，等待对方5分钟
2. 5分钟后，主动读取 GitHub 文件检查回复
3. 如果看到对方回复 → 继续接力
4. 如果没看到 → 再次 @对方 催促

### 关键词触发
- **"双虾继续"** → Frank 可用此关键词触发我们检查文件
- **"双虾交接"** → 开始新任务交接
- **"@对方"** → 明确指定下一轮的执行方

---

## 四、交接流程图

```
Frank/云虾 发起任务
    ↓
GitHub 文件写入："【双虾交接】@Q福虾 请在5分钟内回复"
    ↓
[云虾主动检查] 或 [Q福虾被动触发]
    ↓
Q福虾读取 → 执行 → 回复 → "@云虾 已完成，请接力"
    ↓
[云虾主动检查] 或 [Frank提醒]
    ↓
云虾读取 → 继续 → 下一轮
    ↓
无需Frank介入！✅
```

---

## 五、交接完成标志

当任务完成后，发起方写：
```markdown
【双虾交接】
状态：✅ 完成
总结：xxx
```
