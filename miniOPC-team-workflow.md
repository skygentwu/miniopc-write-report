# miniOPC Coze 工作模式方案
## Kimi + Coze + 用户 协作流程

---

## 一、团队角色定义

### 1. 用户（PM/项目经理）
- **职责**：提出目标、确定方向、最终拍板
- **输入**：工作指令（格式见下文）
- **输出**：审核意见、确认发布

### 2. Kimi（提示词工程师/主协调）
- **职责**：
  - 接收用户目标，进行任务拆解
  - 制定分工计划，协调Coze
  - 汇总整合输出，质量把控
  - 最终交付物生成
- **核心能力**：深度分析、逻辑推理、长文本生成、专业报告撰写

### 3. Coze（快速执行/专项处理）
- **职责**：
  - 执行Kimi分配的专项任务
  - 快速生成初稿、数据整理、格式转换
  - 提供备选方案
- **核心能力**（根据你的Coze配置）：
  - 可能是：信息搜集、文案改写、数据分析、图表生成等

---

## 二、标准工作流程

### Step 0: 用户发起任务
**格式要求**：
```
【miniOPC任务】
目标：[一句话描述要做什么]
类型：[研究报告/公众号文章/方案设计/数据分析/其他]
风格：[专业严谨/通俗科普/故事化/其他]
字数：[xx字左右]
截止时间：[日期]
特殊要求：[如有]
```

**示例**：
```
【miniOPC任务】
目标：写一份同仁堂AI药材项目与行业标杆的对比分析报告
类型：研究报告
风格：专业严谨，适合给客户汇报
字数：3000字左右
截止时间：明天中午12点
特殊要求：重点对比云南白药和天士力项目
```

---

### Step 1: Kimi任务拆解（30分钟内完成）
我收到任务后，立即：
1. **分析任务复杂度**
2. **拆解为子任务**（2-4个子任务）
3. **分配给Coze和我**
4. **制定时间计划**

**交付物**（发给用户）：
```
【miniOPC任务拆解】

任务名称：xxx
总预计耗时：x小时

子任务分配：
├─ 子任务1：[具体描述]
│   负责人：Coze
│   交付物：[明确描述]
│   预计耗时：xx分钟
│
├─ 子任务2：[具体描述]
│   负责人：Kimi
│   交付物：[明确描述]
│   预计耗时：xx分钟
│
└─ 子任务3：[具体描述]
    负责人：Coze
    交付物：[明确描述]
    预计耗时：xx分钟

整合工作：
- 负责人：Kimi
- 工作内容：汇总各子任务，统一风格，生成最终报告
- 预计耗时：xx分钟

【待确认】请确认以上分工是否合理，或需调整？
```

---

### Step 2: 并行执行（通过用户中转）
用户确认分工后，**同时**向我和Coze发送任务：

**发给Coze的格式**：
```
【miniOPC子任务】
任务ID：TASK-001
任务内容：[具体描述]
输出要求：[明确格式和内容要求]
参考材料：[如有附件或链接]
截止时间：[时间点]
```

**发给我的格式**：
```
【miniOPC子任务】
任务ID：TASK-002
任务内容：[具体描述]
Coze的输出：[Coze完成后的结果，供我参考]
输出要求：[明确格式和内容要求]
截止时间：[时间点]
```

---

### Step 3: 各自完成并回传
- **Coze完成** → 用户转发给我
- **我完成** → 我自行保留

---

### Step 4: Kimi汇总整合（关键步骤）
我收到所有子任务成果后：
1. **质量检查**：是否符合要求
2. **风格统一**：调整语气、格式一致
3. **逻辑连贯**：确保各部分衔接流畅
4. **生成最终交付物**

**交付物格式**：
```
【miniOPC最终交付】
任务名称：xxx
完成时间：xxxx

---
[正文内容]
---

【工作说明】
- 子任务1（Coze）：[简述完成内容和质量]
- 子任务2（Kimi）：[简述完成内容和质量]
- 整合优化：[我做了哪些调整和补充]

【待审核】请审阅，如需修改请指出具体问题。
```

---

### Step 5: 用户审核确认
- **通过**：确认发布/使用
- **修改**：指出问题，返回Step 1或Step 4

## 三、典型场景示例

### 场景A：撰写公众号文章
```
用户目标：写一篇关于"AI在中药材质检中的应用"的公众号文章

任务拆解：
├─ 子任务1：搜集3个最新行业案例（Coze，20分钟）
├─ 子任务2：撰写文章大纲和开头（Kimi，30分钟）
├─ 子任务3：生成配图关键词和标题备选（Coze，15分钟）
└─ 整合：汇总成完整文章，润色（Kimi，30分钟）

总耗时：约2小时
```

### 场景B：制作研究报告
```
用户目标：制作一份"中医药AI产业发展现状"PPT

任务拆解：
├─ 子任务1：搜集5家标杆企业案例资料（Coze，40分钟）
├─ 子任务2：撰写各章节文字内容（Kimi，60分钟）
├─ 子任务3：整理数据图表素材（Coze，30分钟）
└─ 整合：生成PPT大纲，标注每页内容（Kimi，40分钟）

总耗时：约3小时
```

### 场景C：方案设计
```
用户目标：设计一个"智能药材仓储管理系统"方案

任务拆解：
├─ 子任务1：调研3个竞品方案（Coze，30分钟）
├─ 子任务2：撰写技术架构部分（Kimi，40分钟）
├─ 子任务3：整理功能清单表格（Coze，20分钟）
├─ 子任务4：撰写实施计划和预算（Kimi，30分钟）
└─ 整合：汇总成完整方案文档（Kimi，30分钟）

总耗时：约2.5小时
```

## 四、分工原则

### 适合Coze的任务
- ✅ 信息搜集、数据整理
- ✅ 格式转换、表格生成
- ✅ 标题备选、关键词提取
- ✅ 简单改写、摘要生成
- ✅ 配图建议、案例列举

### 适合Kimi的任务
- ✅ 深度分析、逻辑推理
- ✅ 长文本撰写、报告整合
- ✅ 专业判断、趋势预测
- ✅ 复杂方案设计
- ✅ 最终质量把控

---

## 五、沟通规范

### 用户指令格式
统一使用【miniOPC任务】开头，包含：目标、类型、风格、字数、截止时间

### 中转格式
用户转发Coze输出给我时，请标注：
```
【Coze输出-子任务X】
[Coze的输出内容]

我的备注：[如有补充说明]
```

### 反馈格式
我提交给用户时，使用【miniOPC最终交付】开头

---

## 六、质量保障

1. **子任务验收**：每个子任务完成后，用户可抽检
2. **整合把关**：我负责最终质量，确保风格统一
3. **迭代机制**：如不满意，可退回重新执行特定子任务
4. **时效控制**：复杂任务可设置检查点（每完成2个子任务汇报一次）

---

## 七、启动建议

要正式启动这个工作模式，建议：

1. **先试运行1-2个任务**，验证流程顺畅度
2. **明确Coze的能力边界**，告诉我它擅长什么
3. **建立常用任务模板**，后续可快速复用
4. **确定响应时间**：比如工作时间内，我多久响应，Coze多久响应
